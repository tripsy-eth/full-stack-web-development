HTML Basics: Essential Tags Explained

Document Structure Tags

Every HTML document starts with these essential structure tags:

  • <!DOCTYPE html> - Declares this as an HTML5 document
  • <html> - Root element that wraps all content
  • <head> - Contains metadata (not visible on page)
  • <body> - Contains all visible page content

Text and Heading Tags

HTML provides six levels of headings:

<h1>This is an H1 heading (largest)</h1>

<h2>This is an H2 heading</h2>

<h3>This is an H3 heading</h3>

<h4>This is an H4 heading</h4>

<h5>This is an H5 heading</h5>
<h6>This is an H6 heading (smallest)</h6>

<p>This is a paragraph tag. It's used for regular text content and automatically adds spacing above and below.</p>

Text Formatting Tags

Here are common ways to format text:

  • <strong>Bold text</strong> using <strong> tag
  • <em>Italic text</em> using <em> tag
  • <u>Underlined text</u> using <u> tag
  • <mark>Highlighted text</mark> using <mark> tag
  • <small>Small text</small> using <small> tag

Block vs Inline Elements

Block Elements (div examples)

Syntax:

<div style="background-color: lightblue; padding: 10px; margin: 5px;">
  This is a div (block element).
</div>

<div style="background-color: lightgreen; padding: 10px; margin: 5px;">
  This is another div.
</div>

Renders as:

Block elements take up the full width available and start on a new line:

This is a div (block element). It takes up the full width and starts on a new line.
This is another div. Notice how it appears below the previous div, even though there's space.
Even when we set width to 50%, this div still starts on a new line.

Inline Elements (span examples)

Syntax:

<p>
  This is a paragraph with
  <span style="background-color: lightcoral; padding: 5px;">inline span elements</span>
  that flow
  <span style="background-color: lightpink; padding: 5px;">within the text</span>
  and don't break to new lines.
</p>

Renders as:

Inline elements only take up as much width as needed and don't start on a new line:

This is a paragraph with inline span elements that flow within the text and don't break to new lines.

Mixed Example

Syntax:

<div style="background-color: #f0f0f0; padding: 10px; margin: 10px;">
  This div contains text and
  <span style="background-color: yellow;">this span stays inline</span>
  with the rest of the text in the div.
</div>

<div style="background-color: #e0e0e0; padding: 10px; margin: 10px;">
  <span style="background-color: orange;">Multiple</span>
  <span style="background-color: cyan;">spans</span>
  <span style="background-color: pink;">in a row</span>
  stay on the same line within this block div.
</div>

Renders as:

Here's what happens when we mix block and inline elements:

This div contains text and this span stays inline with the rest of the text in the div.
Multiple spans in a row stay on the same line within this block div.

Key Differences Summary

  • Block elements (div): Take full width, start on new line, can have width/height set
  • Inline elements (span): Only take needed width, flow with text, width/height ignored

Special HTML Elements

Horizontal Rule (<hr>)

What it does: Creates a horizontal line across the page to separate content sections.

Syntax:

<p>Content above the line</p>
<hr>
<p>Content below the line</p>

Renders as:

Content above the line

Content below the line

Line Break (<br>)

What it does: Forces text to move to the next line without creating a new paragraph.

Syntax:

<p>First line<br>
Second line<br>
Third line</p>

Renders as:

First line
Second line
Third line

Subscript (<sub>) and Superscript (<sup>)

What they do: Make text appear smaller and either below (subscript) or above (superscript) the normal text line. Perfect for chemical formulas, math equations, and footnotes.

Syntax:

<p>Water formula: H<sub>2</sub>O</p>
<p>Einstein's equation: E = mc<sup>2</sup></p>
<p>Mathematical expression: x<sup>2</sup> + y<sub>1</sub> = z<sup>3</sup></p>

Renders as:

Water formula: H2O

Einstein's equation: E = mc2

Mathematical expression: x2 + y1 = z3

Key Points

  • <hr>: Self-closing tag that creates a horizontal line (block element) - takes up full width and starts on new line
  • <br>: Self-closing tag that creates a line break (inline element) - just breaks the line, doesn't add extra spacing
  • <sub>: Makes text appear below the baseline (inline element) - flows with surrounding text
  • <sup>: Makes text appear above the baseline (inline element) - flows with surrounding text

HTML Entities

HTML entities are special codes used to display characters that have special meaning in HTML or characters that are not easily typed on a keyboard. They start with an ampersand (&) and end with a semicolon (;).

Why Use HTML Entities?

  • Reserved Characters: Some characters like <, >, and & have special meaning in HTML
  • Special Symbols: Display symbols like copyright, trademark, or mathematical symbols
  • Non-breaking Spaces: Control spacing and prevent line breaks
  • Accented Characters: Display characters from different languages

Common HTML Entities

Reserved Characters

Syntax and Results:

Character Entity Name Entity Number Result
Less than &lt; &#60; <
Greater than &gt; &#62; >
Ampersand &amp; &#38; &
Quote &quot; &#34; "

Common Symbols

Syntax:

<p>Copyright: &copy; 2024</p>
<p>Trademark: MyBrand&trade;</p>
<p>Registered: MyCompany&reg;</p>
<p>Degree: 25&deg;C</p>
<p>Plus/Minus: &plusmn;5%</p>
<p>Multiplication: 5 &times; 3 = 15</p>
<p>Division: 15 &divide; 3 = 5</p>

Renders as:

Copyright: © 2024

Trademark: MyBrand™

Registered: MyCompany®

Degree: 25°C

Plus/Minus: ±5%

Multiplication: 5 × 3 = 15

Division: 15 ÷ 3 = 5

Spacing Entities

Syntax:

<p>Normal space between words.</p>
<p>Non-breaking&nbsp;space&nbsp;prevents&nbsp;line&nbsp;breaks.</p>
<p>En&ndash;dash (shorter)</p>
<p>Em&mdash;dash (longer)</p>

Renders as:

Normal space between words.

Non-breaking space prevents line breaks.

En–dash (shorter)

Em—dash (longer)

Accented Characters

Syntax:

<p>Caf&eacute; (French)</p>
<p>Ni&ntilde;o (Spanish)</p>
<p>R&eacute;sum&eacute;</p>
<p>Na&iuml;ve</p>

Renders as:

Café (French)

Niño (Spanish)

Practical Example

Problem: You want to display HTML code on your webpage

Wrong way (won't display correctly):

<p>Use <strong> for bold text</p>

Right way (using entities):

<p>Use &lt;strong&gt; for bold text</p>

Renders as:

Use <strong> for bold text

Key Points About HTML Entities

  • Always start with & and end with ;
  • Case-sensitive: &copy; works, but &Copy; doesn't
  • Two formats: Named entities (&copy;) and numbered entities (&#169;)
  • Essential for displaying HTML code: Use &lt; and &gt; to show angle brackets
  • Non-breaking space (&nbsp;): Prevents text from wrapping at that point
  • When in doubt: Use entities for any special characters to ensure they display correctly

Semantic HTML Tags

Semantic HTML uses tags that describe the meaning of content, not just its appearance. This makes your code more accessible, SEO-friendly, and easier to maintain.

Why Use Semantic HTML?

  • Accessibility: Screen readers can better understand your content structure
  • SEO Benefits: Search engines understand your content hierarchy better
  • Code Readability: Other developers can quickly understand your page structure
  • Future-proof: Semantic markup works better with new technologies

Common Semantic Tags

Page Structure Tags:

  • <header> - Top section of page or article (logo, navigation)
  • <nav> - Navigation links
  • <main> - Primary content of the page
  • <article> - Self-contained content (blog post, news article)
  • <section> - Thematic grouping of content
  • <aside> - Sidebar content, related information
  • <footer> - Bottom section (copyright, contact info)

Semantic vs Non-Semantic Example

Non-Semantic (avoid this):

<div class="header">Website Title</div>
<div class="navigation">Menu items</div>
<div class="content">Main content</div>
<div class="sidebar">Related links</div>
<div class="footer">Copyright info</div>

Semantic (better approach):

<header>Website Title</header>
<nav>Menu items</nav>
<main>Main content</main>
<aside>Related links</aside>
<footer>Copyright info</footer>

Text-Level Semantic Tags

  • <time> - Dates and times: January 15, 2024
  • <address> - Contact information for author/owner
  • <blockquote> - Extended quotations from other sources
  • <cite> - References to creative works: The Great Gatsby
  • <code> - Computer code: console.log("Hello");
  • <kbd> - Keyboard input: Press Ctrl+C to copy

Quick Tip

Remember: Choose tags based on meaning, not appearance. You can always style semantic tags with CSS to look however you want, but the semantic meaning helps everyone understand your content better!
